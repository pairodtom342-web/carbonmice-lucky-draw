## วิธีใช้

### ทีม carbonMICE (ก่อนงาน — ที่ admin.html)
1. เข้า **admin.html** ใส่ PIN `2468`
2. กด **+ สร้างงานใหม่** → ใส่ชื่องาน, โลโก้ (ไม่บังคับ), รายการของรางวัล (1 บรรทัด/รางวัล)
3. เพิ่ม **วัน** ได้หลายวัน — แต่ละวันวางรายชื่อของวันนั้นเอง (วางจาก Excel/Sheet, ใส่หน่วยงานคั่น `,` หรือ Tab)
4. กด **💾 บันทึกงาน** → กด **📋 คัดลอก** ลิงก์หน้าบ้าน ส่งให้ลูกค้า
5. หลังงาน: เปิดงานนั้น → **🏆 ดูผล + CSV** → ดาวน์โหลด CSV ผู้ได้รางวัล (แยกตามวัน)

### ลูกค้า / MC หน้างาน (ที่ลิงก์หน้าบ้าน บนจอ LED)
1. เปิดลิงก์ที่ได้รับ → กด **⛶ เต็มจอ**
2. (ถ้ามีหลายวัน) เลือก **แท็บวัน** ด้านบน
3. เลือกของรางวัลที่กำลังจับ (ไม่บังคับ) → กด **🎲 สุ่มรายชื่อ** (หรือ Space)
4. **✅ อยู่ในงาน — รับรางวัล** (ฉลอง 🎉 + บันทึก) หรือ **❌ ไม่อยู่ — ตัดชื่อออก**
5. **↩️ ย้อนกลับ** แก้รายการล่าสุดได้

## จุดเด่น / การทำงานเบื้องหลัง
- **local-first + auto-sync:** จับรางวัลทำงานทันทีบนเครื่อง (ลื่น ไม่รอเน็ต) แล้ว sync ผู้ได้รางวัล "ของวันนั้นทั้งชุด" ขึ้นฐานข้อมูลเบื้องหลัง — เน็ตหลุดจอไม่ค้าง เน็ตกลับมาข้อมูลตามขึ้นเอง (ตัวชี้สถานะมุมขวาบน 🟢/🟡/⚪)
- **แต่ละวันเป็นอิสระ** — คนที่จับไปแล้วถูกตัดออกเฉพาะในวันนั้น (ไม่ผูกข้ามวัน)
- **จำสถานะอัตโนมัติ** — รีเฟรช/จอแครช เปิดใหม่รายชื่อที่จับไปแล้วไม่หาย (localStorage คีย์ = event+วัน)
- ตัวนับสด · รายชื่อผู้ได้รางวัลเรียงล่าสุดบน · confetti + เสียง · คงแบรนด์ carbonMICE (จอมืด คอนทราสต์สูง อ่านง่ายบน LED)

## เทคนิค (ทีมพัฒนา)
- **backend/** = Google Apps Script Web App (แยกไฟล์คนละตัวกับเกมบูท) + Google Sheet
  - Sheet `events` (config ต่อ event) · `winners` (ผู้ได้รางวัลต่อวัน) · โลโก้เก็บเป็นไฟล์ Drive ต่อ event id
  - action: `getEvent`(public) · `syncWinners`(public, replace-per-day) · `listEvents`/`saveEvent`/`deleteEvent`/`getWinners` (ต้องมี ADMIN_KEY)
- **config.js** = `API_URL` (/exec ของ Web App) · `ADMIN_PIN` `2468` · `ADMIN_KEY` (ต้องตรงกับ backend)
- POST ส่งเป็น `text/plain` เลี่ยง CORS preflight · แนวนอน 16:9 (จอตั้งปรับ layout อัตโนมัติ)
- ⚠️ `config.js` เสิร์ฟ public — PIN/ADMIN_KEY จึงไม่ลับสมบูรณ์ (ความเสี่ยงต่ำ = ข้อมูลรายชื่อจับรางวัลบูท) เหมือน posture ของเกมบูท
